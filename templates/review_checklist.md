# Review checklist trước khi nộp báo cáo

Dùng checklist này sau khi đã viết xong bản Word và trước khi xuất PDF.

## 1. Kiểm tra yêu cầu đề bài

| Câu hỏi | Trạng thái |
|---|---|
| Báo cáo có đúng chủ đề giảng viên giao không? | ☐ |
| Đã trả lời đủ các ý tối thiểu trong đề bài chưa? | ☐ |
| Có mở rộng thêm ngoài yêu cầu tối thiểu chưa? | ☐ |
| Báo cáo có tối thiểu 30 trang không? | ☐ |
| Có phần liên hệ thực tế rõ ràng không? | ☐ |
| Có bảng kiểm đối chiếu yêu cầu cuối học phần không? | ☐ |

## 2. Kiểm tra cấu trúc

| Thành phần | Trạng thái |
|---|---|
| Trang bìa | ☐ |
| Trang bìa phụ nếu mẫu yêu cầu | ☐ |
| Mục lục | ☐ |
| Danh mục ký hiệu và viết tắt | ☐ |
| Danh mục hình vẽ | ☐ |
| Danh mục bảng | ☐ |
| Lời cảm ơn | ☐ |
| Lời nói đầu / Mở đầu | ☐ |
| Chương 1 - Cơ sở lý luận | ☐ |
| Chương 2 - Phân tích tình huống | ☐ |
| Chương 3 - Giải pháp và kế hoạch triển khai | ☐ |
| Kết luận | ☐ |
| Tài liệu tham khảo | ☐ |
| Phụ lục | ☐ |

## 3. Kiểm tra bìa

| Câu hỏi | Trạng thái |
|---|---|
| Đã điền họ tên học viên chưa? | ☐ |
| Đã điền mã học viên chưa? | ☐ |
| Đã điền lớp chưa? | ☐ |
| Đã điền tên giảng viên hướng dẫn chưa? | ☐ |
| Không còn placeholder như `[Điền ...]`? | ☐ |
| Tên đề tài đúng chính tả và nhất quán? | ☐ |
| Năm, địa điểm đúng? | ☐ |

## 4. Kiểm tra mục lục, danh mục hình, danh mục bảng

| Câu hỏi | Trạng thái |
|---|---|
| Mục lục đã cập nhật toàn bộ? | ☐ |
| Số trang trong mục lục đúng? | ☐ |
| Không có lỗi số trang kiểu `284`, `415`, `4` bất thường? | ☐ |
| Danh mục hình có đủ số trang? | ☐ |
| Danh mục bảng có đủ số trang? | ☐ |
| Tên hình/bảng trong danh mục khớp với caption trong bài? | ☐ |

## 5. Kiểm tra hình vẽ

| Câu hỏi | Trạng thái |
|---|---|
| Tất cả hình có hiển thị trong PDF? | ☐ |
| Hình đủ rõ, không bị mờ quá mức? | ☐ |
| Mỗi hình có caption đúng định dạng? | ☐ |
| Hình được nhắc tới trong nội dung? | ☐ |
| Không có hình bị trống, mất nội dung hoặc lệch trang? | ☐ |
| Hình workflow/diagram có ý nghĩa phân tích, không chỉ trang trí? | ☐ |

## 6. Kiểm tra bảng

| Câu hỏi | Trạng thái |
|---|---|
| Mỗi bảng có caption đúng định dạng? | ☐ |
| Bảng được nhắc tới trong nội dung? | ☐ |
| Không có bảng bị vỡ, mất cột khi xuất PDF? | ☐ |
| Bảng dài có lặp lại tiêu đề hàng nếu qua trang? | ☐ |
| Tên bảng không nằm cuối trang trong khi bảng trôi sang trang sau? | ☐ |

## 7. Kiểm tra nội dung học thuật

| Câu hỏi | Trạng thái |
|---|---|
| Chương 1 có đủ cơ sở lý luận? | ☐ |
| Chương 2 phân tích nguyên nhân gốc, không chỉ mô tả hiện tượng? | ☐ |
| Chương 3 có giải pháp quản lý, kỹ thuật, đào tạo, truyền thông? | ☐ |
| Có phân tích theo CIA hoặc mô hình phù hợp? | ☐ |
| Có đánh giá rủi ro? | ☐ |
| Có lộ trình triển khai? | ☐ |
| Có chỉ số đánh giá hiệu quả? | ☐ |
| Có liên hệ thực tế đủ cụ thể? | ☐ |

## 8. Kiểm tra tài liệu tham khảo

| Câu hỏi | Trạng thái |
|---|---|
| Tài liệu tham khảo có đánh số `[1]`, `[2]`, `[3]`? | ☐ |
| Trong thân bài có trích dẫn tương ứng? | ☐ |
| Nguồn pháp lý, chuẩn kỹ thuật được ghi đúng tên? | ☐ |
| Không dùng nguồn không đáng tin cậy nếu có nguồn chính thống? | ☐ |
| Các văn bản pháp lý mới đã kiểm tra lại hiệu lực? | ☐ |

## 9. Kiểm tra ngôn ngữ

| Câu hỏi | Trạng thái |
|---|---|
| Không còn lỗi chính tả rõ ràng? | ☐ |
| Thuật ngữ dùng nhất quán? | ☐ |
| Không dùng văn phong quá khẩu ngữ? | ☐ |
| Không có đoạn lặp quá nhiều? | ☐ |
| Các câu kết luận đủ thận trọng, không khẳng định quá mức? | ☐ |

## 10. Kiểm tra xuất file

| Câu hỏi | Trạng thái |
|---|---|
| Đã lưu bản Word cuối cùng? | ☐ |
| Đã xuất PDF? | ☐ |
| PDF mở được trên máy khác? | ☐ |
| Số trang PDF đúng yêu cầu? | ☐ |
| Tên file rõ ràng? | ☐ |

Tên file đề xuất:

```text
Nguyen_Ngoc_Son_CHAT4P016_QLATTT_De_13.pdf
```

Hoặc dùng mẫu chung:

```text
[HoTen]_[MaHocVien]_[MonHoc]_[DeTai].pdf
```
